Для работы системы обновления основного проекта необходимо скомпилировать QtAutoUpdater.
Итоговые файлы:
  Qt5AutoUpdaterCore.dll
  Qt5AutoUpdaterWidgets.dll
  plugins\updaters\qtifw.dll

Для работы основного приложения любого типа (Debug, Release) достаточно библиотек QtAutoUpdater типа Release.
Cборка проверена с Qt 5.15.

1. Если при клонировании репозитория основного проекта не была указана опция --recurse-submodules:
  git submodule update --init --recursive
2. Сборка
  qmake
  mingw32-make -j<n>
    где <n> — число ядер ЦП
2а. Сборка в Qt Creator
  - Открыть qtautoupdater.pro
  - при первом открытии будет предложено настроить комплект (Kit) и тип сборки (Debug, Release)
  - для соответствующего комплекта выбрать желаемый тип сборки (кнопка [Details ▼])
    • для комплекта по умолчанию список типов сборки будет развёрнут
    • если вы не знаете в чем разница, выбирайте Release
  - путь изменять не нужно
  - нажмите кнопку [Configure Project] под этим списком
  - Выключите режим Shadow build перед сборкой:
    а) Projects на левой панели (или Ctrl+5)
    б) в группе General убрать галочку Shadow build (для каждого комплекта и для каждой конфигурации сборки индивидуально).
  - В меню Build выберите Build project ... (или нажмите Ctrl+B)
    • в конце сборки итоговые библиотеки будут автоматически скопированы в директорию lib основго проекта (т. е. здесь больше ничего делать не нужно)
